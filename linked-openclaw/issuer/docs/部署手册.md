# Issuer 部署手册

## 1. 适用范围

这份手册面向一台 Linux 机器，从零部署当前目录下这套 `Issuer + OpenClaw Gateway`。

系统分工：

- Feishu 侧负责接收用户请求和返回草案 / 执行结果
- OpenClaw 负责会话、模型调用、hook 装载和技能执行
- GitHub 工具负责真正的 issue create / update / close / comment

部署目录不是固定的。下面统一用 `APP_DIR` 表示你的实际部署目录。

例如：

```bash
export APP_DIR=/opt/deploy/issuer
```

默认端口：

- `issuer` 本地 gateway：`18890`

## 2. 机器前置条件

建议环境：

- Ubuntu 22.04 / 24.04
- 能使用 `sudo`
- 已安装 `systemd`
- 机器可以访问：
  - `api.github.com`
  - `https://open.feishu.cn`
  - `https://router.yeying.pub`

先装基础工具：

```bash
sudo apt update
sudo apt install -y \
  git curl jq ca-certificates unzip \
  openssh-client build-essential \
  python3
```

## 3. 安装 `openclaw`

本项目使用仓库内维护的 OpenClaw 配置：

- `config/openclaw.json`
- `data/openclaw/state/`

安装 CLI 后，直接使用仓库内配置即可。

推荐方式一：官方安装脚本

```bash
curl -fsSL https://openclaw.ai/install.sh | bash -s -- --no-onboard
openclaw --version
```

推荐方式二：如果你已经自己管理 Node/npm

```bash
npm install -g openclaw@latest
openclaw --version
```

注意：

- 本仓库最终调用的是 `scripts/openclaw-local`
- OpenClaw CLI 安装完成后，无需额外执行 `openclaw onboard`

## 4. GitHub 侧准备

### 4.1 GitHub App

需要准备一个 GitHub App，并安装到目标仓库。

这个 App 至少要能完成这些动作：

- 读取 issue 与标签
- 创建 issue
- 更新 issue
- 关闭 issue
- 在 issue 下发评论

### 4.2 GitHub App 私钥

把 GitHub App 下载到的私钥放到：

```bash
mkdir -p "$APP_DIR/secrets"
chmod 700 "$APP_DIR/secrets"
```

目标文件：

```bash
$APP_DIR/secrets/github-app.pem
```

建议权限：

```bash
chmod 600 "$APP_DIR/secrets/github-app.pem"
```

## 5. Feishu / OpenClaw 侧准备

### 5.1 Feishu 应用

你需要一个能被 OpenClaw gateway 接入的飞书应用，并拿到：

- `appId`
- `appSecret`
- 目标群 `chat_id`

至少确认：

- 应用已被拉进目标群
- 事件订阅走长连接
- 应用具备群消息接收权限

### 5.2 OpenClaw 静态配置

复制模板：

```bash
cd "$APP_DIR"
cp config/openclaw.json.template config/openclaw.json
```

至少替换这些值：

- `models.providers.router.apiKey`
- `channels.feishu.appId`
- `channels.feishu.appSecret`
- `channels.feishu.groupAllowFrom`
- `gateway.auth.token`

这条很关键：

- `messages.groupChat.visibleReplies` 建议显式设为 `automatic`

否则群里可能出现“处理了但没有可见回复”。

## 6. 拉代码与初始化配置

```bash
mkdir -p "$(dirname "$APP_DIR")"
git clone <your-repo-url> "$APP_DIR"
cd "$APP_DIR"
```

建议先建目录：

```bash
mkdir -p data/logs data/openclaw/state data/openclaw/workspace-larkbot secrets
```

## 7. 填写配置

### 7.1 GitHub App 配置

编辑：

```bash
vim "$APP_DIR/config/github-app.config.env"
```

最关键的字段：

```env
GITHUB_APP_ID=
GITHUB_APP_PRIVATE_KEY_PATH=secrets/github-app.pem
GITHUB_APP_INSTALLATION_ID=
GITHUB_DEFAULT_OWNER=yeying-community
GITHUB_DEFAULT_REPO=robot
```

说明：

- `GITHUB_APP_INSTALLATION_ID` 可以留空，工具会按 owner/repo 自动查询
- `GITHUB_APP_PRIVATE_KEY_PATH` 是相对 `APP_DIR` 解析的
- 如果你直接想用 token，也可以写 `GITHUB_TOKEN=...`

### 7.2 OpenClaw 配置

编辑：

```bash
vim "$APP_DIR/config/openclaw.json"
```

至少检查这些字段：

- `agents.defaults.workspace`
- `models.providers.router.apiKey`
- `channels.feishu.appId`
- `channels.feishu.appSecret`
- `channels.feishu.groupAllowFrom`
- `gateway.port`
- `gateway.auth.token`
- `messages.groupChat.visibleReplies`

推荐配置：

- `agents.defaults.workspace=$APP_DIR/data/openclaw/workspace-larkbot`
- `gateway.port=18890`
- `messages.groupChat.visibleReplies=automatic`

### 7.3 策略配置

编辑：

```bash
vim "$APP_DIR/config/policy.json"
```

这几个字段最关键：

```json
{
  "admins": [
    "ou_xxx_replace_with_admin_user_id"
  ],
  "repoAliases": [
    {
      "alias": "robot",
      "owner": "yeying-community",
      "repo": "robot"
    }
  ]
}
```

## 8. 首次自检与一键安装

推荐直接跑：

```bash
cd "$APP_DIR"
./scripts/bootstrap.sh
```

`bootstrap.sh` 会做这些事：

- 创建 `config/`、`secrets/`、`data/`
- 生成缺失的配置文件
- 同步 `workspace_assets/` 到运行时 workspace

首次启动：

```bash
cd "$APP_DIR"
./scripts/start_gateway.sh
./scripts/status_gateway.sh
```

## 9. systemd 部署与重启

安装服务：

```bash
cd "$APP_DIR"
BOT_USER="$(id -un)" ./scripts/install_systemd.sh
```

重启并检查：

```bash
sudo systemctl restart issuer-openclaw-gateway
sudo systemctl status --no-pager issuer-openclaw-gateway
journalctl -u issuer-openclaw-gateway -n 100 --no-pager
```

## 10. 运行时目录说明

- `config/`：你手工维护的静态配置
- `workspace_assets/`：源码态的 tools / hooks / skills
- `data/openclaw/workspace-larkbot/`：运行时真正执行的 workspace
- `data/openclaw/state/`：OpenClaw 自己的状态目录
- `data/logs/openclaw-gateway.log`：gateway 日志
- `data/logs/issuer-audit.jsonl`：issuer 审计日志

如果你改的是 `workspace_assets/`，一定记得：

```bash
./scripts/sync_workspace.sh
./scripts/stop_gateway.sh
./scripts/start_gateway.sh
```

## 11. 常见排查

如果飞书里完全没反应，优先检查：

- `config/openclaw.json` 里的飞书 `appId` / `appSecret`
- `groupAllowFrom` 是否包含当前群的 `chat_id`
- 当前消息是否真的 `@issuer`
- `./scripts/status_gateway.sh` 的最近日志

如果有处理但群里没回消息，优先检查：

- `config/openclaw.json` 里的 `messages.groupChat.visibleReplies`
- 当前是否显式用了 `automatic`
- 日志里是否有 `dispatch complete (... replies=0)`

如果整体反应很慢，优先检查：

- 日志里是否有 `llm-idle-timeout`
- 日志里是否有 `Concurrency limit exceeded for account`
- 日志里是否有 `internal_server_error`
- 当前模型是否仍是 `router/gpt-5.4`

如果 GitHub 返回 401 或 403，优先检查：

- GitHub App 是否真的安装到了目标仓库
- `Issues` 权限是否为 `Read and write`
- `config/github-app.config.env` 里的 owner/repo 是否填对
- `secrets/github-app.pem` 是否与这个 App 对应
- 如果手填了 `GITHUB_APP_INSTALLATION_ID`，它是否属于同一个安装
